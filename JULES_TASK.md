# Jules Supervisor Task — AGENTS_RULES v2

> **Роль:** Ты — супервизор базы знаний AGENTS_RULES-v2. Твоя задача — проверять здоровье репозитория, исправлять проблемы и задавать вопросы пользователю когда неуверен.

---

## Автономный режим (каждый час)

Ты запускаешься автоматически через GitHub Actions (`.github/workflows/jules-monitor.yml`).

### Шаг 1: Синхронизация
```bash
cd /home/segun/CascadeProjects/Перед\ 0_2/AGENTS_RULES-v2
git pull --rebase origin main 2>/dev/null || git pull origin main
```

### Шаг 2: Механическая проверка + автоисправление
```bash
python3 supervisor.py --fix
```

### Шаг 3: Генерация вопросов
```bash
python3 supervisor.py --jules-mode
```
В блоке `--- QUESTIONS FOR USER ---` будут вопросы, которые нужно задать. Если вопросы есть — **создай комментарий в GitHub Issue** или новый Issue с меткой `question`.

### Шаг 4: Интеллектуальная проверка

1. **Прочитай `shared_status.md`** — актуален ли статус?
2. **Прочитай `index.md`** — все ли ссылки валидны?
3. **Просмотри `wiki/errors/`** — если ошибка уже исправлена в коде, добавь:
   ```
   > ✅ ИСПРАВЛЕНО (YYYY-MM-DD): краткое описание что исправлено
   ```
4. **Просмотри `wiki/skills/`** — если навык устарел, добавь:
   ```
   > ⚠️ УСТАРЕЛО (YYYY-MM-DD): причина
   ```
5. **Проверь `raw/`** — все ли логи содержат обязательные секции

### Шаг 5: Компиляция (если есть новые логи)
```bash
python3 compile.py --no-git
```

### Шаг 6: Закоммить
```bash
git add -A
git commit -m "[Jules-Supervisor] health check: описание что сделано"
git push origin main
```

## 🔴 Как задавать вопросы пользователю

Jules не может вести чат. Но может создавать **GitHub Issues** с вопросами:

### Когда создавать Issue с вопросом:
- Нашёл ошибку, но не знаешь как её исправить
- Устаревший навык, но не знаешь чем заменить
- shared_status.md противоречит wiki/
- Нашёл contradicting записи в wiki
- Не понятно, какой агент работал последним

### Формат Issue с вопросом:
```markdown
**Label:** question, jules-task

# ❓ Вопрос от Jules-Supervisor

## Контекст
[Что ты нашёл и где]

## Вопрос
[Конкретный вопрос к пользователю]

## Варианты ответа
1. [Вариант 1]
2. [Вариант 2]
3. Свой вариант

> Ответь комментарием к этому Issue — Jules подхватит при следующем визите.
```

### Когда НЕ создавать Issue:
- ✅ Можешь исправить сам → просто исправь и закоммить
- ✅ Это ложная ошибка → `supervisor.py --fix` удалит автоматически
- ✅ Это дубликат → `supervisor.py --fix` удалит автоматически

---

## Критерии принятия решений

| Ситуация | Действие |
|----------|----------|
| Ошибка в wiki/errors/ уже исправлена в коде | Добавь `> ✅ ИСПРАВЛЕНО` + дата |
| Навык в wiki/skills/ устарел | Добавь `> ⚠️ УСТАРЕЛО` + причина |
| Страница wiki > 15KB | Разбей на две страницы |
| shared_status.md > 2KB | Сократи |
| Лог без «Что НЕ получилось» | Запиши в отчёт |
| Не знаешь что делать | Создай Issue с вопросом |

## НЕ делай

- НЕ удаляй записи целиком (только помечай)
- НЕ меняй `rules_general.md` или `session_handover.md` без причины
- НЕ трогай файлы рабочего проекта
- НЕ запускай `compile.py --recompile` без необходимости

## Промпт-инженерия

### Принципы
1. **Few-Shot через system-промпт** — Эталоны из KB вставляются как «ПРИМЕРЫ ЖЕЛАЕМЫХ ОТВЕТОВ» в `role: system`. НЕ использовать фейковые диалоги role:user/assistant.
2. **Качество промпта > количество текста** — с KB модель получает МЕНЬШЕ текста (4000 vs 5000 символов), но точнее.
3. **Ограничение входного текста** — `text[:6000]` для защиты контекстного окна. С KB: `text_limit = 4000`.

### CoT промпт (V6, 7 шагов)
```
ШАГ 1: ОПРЕДЕЛИ ТИП ДОКУМЕНТА
ШАГ 2: НАЙДИ ЗАКАЗЧИКА (маркеры: "Заказчик:", "Администрация")
ШАГ 3: НАЙДИ РАЗРАБОТЧИКА (маркеры: "ИП", "ООО")
ШАГ 4: ИЗВЛЕКИ ГОД (шифр /25 → 2025)
ШАГ 5: СОБЕРИ НАЗВАНИЕ
ШАГ 6: ОПИШИ СОДЕРЖАНИЕ
ШАГ 7: СФОРМУЛИРУЙ ЦЕЛЬ
```

### XML Structure Hints
Для XML-документов добавлено поле `_xml_structure_hints` в KB:
- ТЗ: TypeCode=05.03, поиск в `<Designers>`, а не `<Author>`
- ПЗ: `<IssueAuthor>` для разработчика
- Игнорировать имена людей (ГИП, директор)

### Обязательные правила в промпте
- `"ВСЕ ЗНАЧЕНИЯ в JSON должны быть СТРОГО НА РУССКОМ ЯЗЫКЕ"`
- `"Нам нужно ЮРИДИЧЕСКОЕ ЛИЦО"` (не ФИО)
- `"ТРЕБУЕТСЯ ДЕТАЛЬНЫЙ МАРКИРОВАННЫЙ СПИСОК"` для content_summary

## Маршрутизация моделей (планируется)
| Тип документа | Модель | Причина |
|--------------|--------|---------|
| Default | Gemma 4-31B | Лучше по среднему (77.5% vs 68.2%) |
| ТЗ, ТУ, Доверенности | Mistral 14B | 100% точность на этих типах |
| Scanned PDF | Gemma + OCR | Mistral 0% на сканах без OCR |

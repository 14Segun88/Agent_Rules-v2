## Проблема 1: One-Shot Overfitting
При добавлении в промпт одного фейкового примера `{customer: "ООО Ромашка"}`, модель начинает возвращать `"ООО Ромашка"` на **ВСЕ** документы.

**Решение:** Исключить выдуманные имена из диалогов. Загружать только подлинные примеры из KB в `role: system`.

---

## Проблема 2: Ответы на английском
При указании ключей на английском (`title`, `customer`), модель переводит и **значения** на английский.

**Решение:** Жёстко прописывать:
```
"ВСЕ ЗНАЧЕНИЯ в JSON должны быть СТРОГО НА РУССКОМ ЯЗЫКЕ. Никакого перевода на английский!"
```

---

## Проблема 3: Копирование шаблонов
Модель копировала шаблон document_type `"Текстовый и графический документ"` вместо определения реального типа.

**Решение:**
- НЕ зашивать текстовые шаблоны в промпт
- Требовать маркированный список для `content_summary`
- Запрещать писать имена людей: `"Нам нужно ЮРИДИЧЕСКОЕ ЛИЦО"`

---

## Проблема 4: Model Unloaded
LM Studio возвращает `Model unloaded` при указании точного имени модели, если модель не загружена.

**Решение:** ~~Использовать `"local-model"`~~ — **НЕ РАБОТАЕТ!** Всегда указывать актуальное имя из `curl http://192.168.47.22:1234/v1/models`.

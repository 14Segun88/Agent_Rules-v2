## Проблема 1: Boost по разделу игнорирует шифр проекта
Алгоритм `_calculate_score()` устанавливал `score=0.99` при совпадении раздела, игнорируя шифр проекта.

**Пример:** Файл `"Раздел_ПД№9_МКД-0109-2024-ПИР-ПБ"` получал score=0.99 с KB записью `"Раздел 9 (МЕС-БМК-04/24-ПБ)"`, хотя шифры разные.

**Решение:**
```python
# Если раздел И шифр совпадают → score=0.99
# Если только раздел → score=0.70 (ниже приоритет)
```

**Паттерны шифров:**
- `МКД-0109-2024`
- `МЕС-БМК-04/24`, `МЕС-БМК-04_24`, `МЕС-БМК-04.24`
- `157/25`, `04/24`

> ✅ ИСПРАВЛЕНО (2026-04-13)

**Решение:** Скрипт `remove_kb_duplicates.py` + резервная копия `knowledge_base_backup.json`.

## Проблема 3: Пересчёт score без boost'ов
`_get_kb_direct_fields()` пересчитывал score без boost'ов → KB override не срабатывал.
> ✅ ИСПРАВЛЕНО (2026-04-13)

**Решение:** Вынесен единый метод `_calculate_score()`, используемый ВЕЗДЕ.

## Проблема 4: Неверные эталоны для другого пакета
KB содержит эталоны Пакета 1 (МЕС-БМК/МКД) → применяются к документам Пакета 2 (СОШ 10) = ЛОЖНЫЕ FAIL.

**Решение:** Добавить ~50 эталонов для СОШ 10 через `add_kb_pkg2.py`.

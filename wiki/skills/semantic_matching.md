## Суть
Метод `_get_semantic_examples()` подбирает наиболее релевантные записи из KB по имени файла, используя токенный matching вместо простого строкового сравнения.

## Механизм matching

### Boost по ключевым словам
| Ключевое слово | Score |
|---------------|-------|
| АР, КР, ПБ, ТЗ, ИОС, ПОС, ОДИ | 75% |
| ИОС5.3 + 5.5.3 | 95% |
| ИОС5.3 + ИОС5 | 90% |

### Boost по разделу + шифру
```python
# Если раздел И шифр совпадают → score = 0.99
# Если только раздел → score = 0.70
```

### Извлечение шифров
```python
patterns = [
    r'МКД-\d{4}-\d{4}',       # МКД-0109-2024
    r'МЕС-БМК-\d{2}[/_\.]\d{2}',  # МЕС-БМК-04/24
    r'\d{3}/\d{2}',             # 157/25
]
```

### Нормализация
Все шифры нормализуются: `/` → `-`, `_` → `-` для корректного сравнения.

## Пример
Файл: `"Раздел_ПД№5_подраздел_ПД№5_часть3_МКД-0109-2024-ПИР-ИОС5.3"`
- Без boost: score = 0.41 (с записью ИОС1 - электроснабжение)
- С boost: score = 0.95 (с записью ИОС5.3 - видеонаблюдение) ✅

## 📌 Главный принцип

**Этот репозиторий — единый источник истины для всех агентов.** Любые знания, решения, ошибки и статусы фиксируются здесь. Агенты НЕ полагаются на собственную внутреннюю память между сессиями.

## 🔄 Жизненный цикл сессии

Каждый агент при работе обязан следовать протоколу из `session_handover.md`:

1. **Check-in** → Прочитать `shared_status.md` + релевантные wiki-страницы.
2. **Работа** → Выполнять задачу, фиксировать промежуточные логи в `raw/`.
3. **Check-out** → Обновить `shared_status.md`, записать результат в `raw/`.

---

## ⚙️ Стандарт кода (Python)

- **Type Hints** — обязательны для всех функций.
- **PEP 8** — обязательный стандарт форматирования.
- **Логирование:** Использовать модуль `logging` (уровень INFO по умолчанию). **НЕ** использовать `print()` в финальном коде.
- **Docstrings:** Обязательны для всех публичных функций и классов (Google-стиль).

## 🗄️ Хранение данных (КРИТИЧНО)

### Правило «Успех / Неудача»
Каждый агент **ОБЯЗАН** в конце задачи записать в `raw/`:
- ✅ **Что получилось** — конкретные результаты с метриками.
- ❌ **Что НЕ получилось** — ошибки, тупики, отмётки. **Это предотвращает повторение ошибок будущими агентами.**

### Структура знаний
| Директория | Назначение | Размер |
|------------|-----------|--------|
| `shared_status.md` | Текущий статус проекта | **max 2KB** |
| `raw/` | Сырые логи сессий (append-only) | Неограниченно |
| `wiki/` | Компилированная база знаний | По страницам |
| `wiki/errors/` | Категоризированные ошибки | По теме |
| `wiki/skills/` | Успешные решения и приёмы | По теме |

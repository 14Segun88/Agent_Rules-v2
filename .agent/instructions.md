## 🔴 АВТОМАТИЧЕСКИЙ CHECK-OUT (выполни ПЕРЕД завершением)

После завершения задачи или при переходе в режим ожидания:

### Шаг 1: Создай лог сессии
Создай файл в `AGENTS_RULES-v2/raw/` с именем:
```
ГГГГ-ММ-ДД_ЧЧ-ММ_<ИмяАгента>.md
```

Пример: `2026-04-11_22-00_Antigravity.md`

Содержимое файла — **ОБЯЗАТЕЛЬНАЯ СТРУКТУРА:**
```markdown
# Лог сессии [Дата] [Агент]

## Задача
Краткое описание задачи.

## Что получилось ✅
- Конкретный результат 1
- Конкретный результат 2 (с метриками если есть)

## Что НЕ получилось ❌
- Проблема 1: описание + почему не вышло
- Проблема 2: описание
(Если всё получилось — напиши «Всё по плану, без блокеров»)

## Технические выводы
- Вывод 1 (для будущих сессий)
- Вывод 2

## Следующий шаг
Чёткая формулировка: что должен сделать следующий агент.

## Блокеры
- Блокер 1 (если есть, иначе «Нет»)
```

### Шаг 2: Обнови shared_status.md
**ЗАМЕНИ** (не дополни!) содержимое `AGENTS_RULES-v2/shared_status.md` актуальным кратким статусом. Файл должен быть **< 2KB**.

### Шаг 3: Закоммить и запушь
```bash
cd /home/segun/CascadeProjects/Перед\ 0_2/AGENTS_RULES-v2
git add -A
git commit -m "[ИмяАгента] checkout: краткое описание"
git push origin main
```

Компиляция `raw/ → wiki/` произойдёт автоматически через `watcher.py`.

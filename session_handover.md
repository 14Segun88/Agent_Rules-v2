# Протокол передачи сессий v2 (Handover Protocol)

> Этот документ **обязателен для исполнения** всеми агентами при начале и завершении работы.

---

## 📥 Check-in (Начало сессии)

Перед выполнением **любой** задачи агент обязан:

### 1. Синхронизировать репозиторий
```bash
cd /home/segun/CascadeProjects/Перед\ 0_2/AGENTS_RULES-v2
git pull --rebase origin main 2>/dev/null || git pull origin main
```

### 2. Прочитать текущий статус
Прочитать **ТОЛЬКО** `shared_status.md` (~2KB). Этого достаточно для понимания текущего состояния проекта.

### 3. Загрузить релевантный контекст
На основе текущей задачи — прочитать **только нужные** wiki-страницы:
- Задача связана с LLM? → `wiki/llm_integration.md`
- Возникла ошибка? → `wiki/errors/` (искать по теме)
- Нужно решение? → `wiki/skills/` (искать по теме)
- Нужна общая картина? → `index.md`

> ⚠️ **НЕ** читать все файлы подряд. Каждый файл = токены. Читай только то, что нужно для текущей задачи.

### 4. Проверить последние изменения в рабочем проекте
```bash
cd /home/segun/CascadeProjects/Перед\ 0_2
git log --oneline -n 5
```

---

## 📤 Check-out (Завершение сессии)

Перед окончанием работы или переходом в режим ожидания:

### 1. Создать лог сессии в `raw/`
Создать файл `raw/YYYY-MM-DD_HH-MM_<AgentName>.md` с содержимым:
```markdown
# Лог сессии [Дата] [Агент]

## Задача
Краткое описание задачи.

## Что получилось ✅
- Конкретный результат 1
- Конкретный результат 2 (с метриками)

## Что НЕ получилось ❌
- Проблема 1: описание + почему не вышло
- Проблема 2: описание

## Технические выводы
- Вывод 1 (для будущих сессий)
- Вывод 2

## Следующий шаг
Чёткая формулировка: что должен сделать следующий агент.

## Блокеры
- Блокер 1 (если есть)
```

### 2. Обновить `shared_status.md`
Заменить содержимое файла (не дополнить, а **заменить**) актуальным статусом. Файл должен быть **максимально кратким (< 2KB)**.

### 3. Зафиксировать и отправить изменения
```bash
cd /home/segun/CascadeProjects/Перед\ 0_2/AGENTS_RULES-v2
git add -A
git commit -m "[AgentName] checkout: краткое описание"
git push origin main
```

## 🕒 Формат записи статуса

```
[ГГГГ-ММ-ДД ЧЧ:ММ] [Агент: Имя] СТАТУС: Текст статуса. СЛЕДУЮЩИЙ ШАГ: Описание.
```

**Пример:**
```
[2026-04-11 22:00] [Агент: Antigravity] СТАТУС: Завершён бенчмарк Gemma vs Mistral. СЛЕДУЮЩИЙ ШАГ: Добавить эталоны Пакета 2 в KB.
```

---

## 🔁 Автоматическая компиляция

После Check-out, файлы в `raw/` автоматически обрабатываются `watcher.py` → `compile.py`:
- Извлекаются **ошибки** → `wiki/errors/`
- Извлекаются **skills** → `wiki/skills/`
- Обновляется `index.md`

> Агенту **НЕ нужно** вручную обновлять wiki/. Это делает компилятор.
